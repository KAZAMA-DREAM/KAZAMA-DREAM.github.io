Java习题：Java中子类和父类相关方法的执行顺序

解析：

1、main中new B()创建对象，然后调用该对象的getValue()方法。

2、执行B的构造方法，B的构造方法中调用了super方法，即执行B的父类A的有参构造方法。

3、A的有参构造方法中调用setValue(v)方法，但是此时A类和B类都有一个setValue()方法，那应该执行哪一个？其实它执行的是B类中的setValue()方法。显然这需要巩固一个知识点：当子类重写了父类的函数，那么子类的对象如果调用该函数，一定调用的是重写过后的函数。可以通过super关键字进行父类重写函数的调用。因为现在正在执行的是B类的构造方法，所以默认先会调用B类中的方法，如果B类中没有，才会调用其父类A中的方法。

4、接下来执行到super.setValue(2*value)，即执行A类的setValue()方法，这时A类的成员变量value就变成了10。

5、这时B类的构造方法中的super(5)就执行完了，然后就到了setValue(getValue()-3)方法。

6、接着执行getValue()方法，首先在B类中找，但B类没有getValue()方法，所以就执行A类中的getValue()方法，A类中的getValue()方法中有try-catch-finally块。先让A的成员变量value自增，从之前的10变为了11，然后直接返回value，没有捕获异常，继续到finally里的this.setValue(value)。

7、然后这个this指的是A类还是B类，答案是B类，因为现在是在执行B的构造方法，所以this指的应该是B类，即调用B类的setValue(int value)方法。

8、然后又super.setValue(2value)；执行父类A的setValue(int value)，把211作为参数传递，A类的setValue(int value)把传进来的value值赋给了A的成员变量value，变成了22。

9、然后this.setValue(value)就执行完了，最后输出value，22。

10、到这儿getValue()方法就执行完了，但是有一点要注意，此时的value值为22，但是getValue()的返回值是11，因为在try{}已经return了，所以这个方法的返回值已经保存下来了，是11，即使finally{}里面又对value的值做出了改变，但是getValue()的返回值是不会变的（除非finally里也有return返回值，它会覆盖前面try的返回值）。接着继续执行B类构造方法中的setValue(getValue()-3)；getValue()是11，所以B的setValue(int value)方法的参数就是8，接着又到了super.setValue(2*value)。

12、调用A类中的setValue(int value)方法，同时将参数赋值给A类的成员变量value，此时value变为16。

13、到这儿B类的构造方法就都执行完了，也就是new B()，然后又调用了该对象的getValue()方法，B类没有，但是父类A有。

14、继续try-catch-finally，A类的成员变量value为16，然后value++再返回，这时getValue()的返回值已经确定了，就是17，即使在finally中对value做出改变，其返回值不会变。然后到finally，又是setValue(value)，调用B类中的setValue(int value)。

15、接着又是super.setValue(2*value)，调用A类的setValue()，并把2 * 17作为参数传递过去。

16、把参数赋值给A的成员变量value，这时this.setValue(value)就执行完了，此时的value为34。最后输出value。

17、需要注意的是，此时的getValue()方法的返回值是17，这个前面已经提到了，到这儿，整个new B().getValue()就执行完了，最后又输出流getValue的返回值，也就是17。所以整个过程执行完后的输出结果是22、34、17。

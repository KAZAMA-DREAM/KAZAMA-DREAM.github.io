Mac笔记本安装和配置maven

一、通过brew安装maven

1、如果你的macbook安装了homebrew，使用brew工具去搜索maven包

2、使用info查看maven包当前的信息情况，包括版本依赖描述等

3、如果你的macbook安装了homebrew，可以用如下命令安装。可以去homebrew官方网站查看maven安装包的详细信息。https://formulae.brew.sh/formula/maven 可以看到我们jdk已经装过了（如果mac上jdk没有安装和配置，看这篇博客），没问题，直接安装maven。

4、安装完后可以使用命令brew list maven来查看maven的安装位置


5、安装过程很顺利，这时候brew就已经帮我们做好了环境变量了。很多教程在这一步后会手动去声明maven的目录情况，我看了几个教程里面的设置完全没有道理，根本链接的不是brew的安装位置。如果我们已经配置好jdk环境，那么执行以下命令会有效果，否则会报缺少jdk配置的错误

6、需要注意的是 /opt/homebrew/Cellar/maven/3.6.3_1/libexec/conf/ 这个目录，这个目录下面存放了maven的配置文件


二、配置Maven

1、查看所安装的Maven中settings.xml文件所在位置，因为要在此配置文件中修改中央仓库、本地仓库等位置信息，不然maven下载包的速度很慢

2、maven中央仓库在国外，为了提高使用体验可修改conf目录下的配置文件settings.xml，打开配置文件找到mirrors节点，在mirrors节点下添加子节点。

3、修改完成后，不要忘记把repository路径也改一下，然后保存即可。


三、IDEA的Maven配置

1、我们打开idea的偏好设置，搜索maven，出现以下的配置界面。


2、我们将安装目录和配置目录替换成配置maven时确定的路径，不好通过访达选文件就直接把路径复制填上去，应用保存即可。



四、maven常用命令

  • 查看maven的版本： mvn -v
  • 编译源代码： mvn compile
  • 编译测试代码：mvn test-compile
  • 运行测试：mvn test
  • 产生site：mvn site
  • 打包：mvn package
  • 在本地Repository中安装jar：mvn install
  • 清除产生的项目：mvn clean
  • 生成eclipse项目：mvn eclipse:eclipse
  • 生成idea项目：mvn idea:idea
  • 组合使用goal命令，如只打包不测试：mvn -Dtest package
  • 编译测试的内容：mvn test-compile
  • 只打jar包: mvn jar:jar

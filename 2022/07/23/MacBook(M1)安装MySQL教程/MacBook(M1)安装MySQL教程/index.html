MacBook(M1)安装MySQL教程

一、下载MySQL社区版本

1、选择你想要下载的版本，为了安装更方便，建议下载dmg安装包，最新版本为8.0.29，m1芯片的记得选ARM版本


2、点击Archives还可以选择历史版本，选择自己想要的！（建议还是用最新的，因为新版本原生支持ARM芯片）



二、安装MySQL社区版本

1、双击下载的dmg文件，加载镜像，开始安装。


2、一直点击继续就可以安装成功。


三、启动MySQL

1、打开系统偏好设置，会发现多了一个mysql图标，点击它，会进入MySQL的设置界面。

2、安装之后默认MySQL的状态是stopped，关闭的，需要点击 Start MySQL Server 按钮来启动它，启动之后，状态会变成running。下方还有一个复选框按钮，可以设置是否在系统启动的时候自动启动MySQL，默认是勾选的，建议取消，节省开机时间。



四、配置MySQL环境变量

1、编辑.bash_profile文件

2、在环境变量中添加mysql的安装路径

3、刷新环境变量

4、终端测试mysql，输入密码

5、mysql安装成功


6、此时关闭终端，重新打开，输入 mysql -u root -p ，会显示 mysql: command not found ，此时再次执行上面第3步的命令，并再次输入 mysql -u root -p 就可以使用。这说明你每次关闭终端后都要source一下才能使用。出现这个问题发原因是：mac默认启动执行的脚本变为了 ~/.zshrc，zsh加载的是~/.zshrc文件，而~/.zshrc文件中并没有定义任务的环境变量（即不会加载~/.bash_profile中的配置，导致不生效）

解决方案如下：

如果你发现自己没有 ~/.zshrc这个文件也没有关系，执行以下命令新建一个


在~/.zshrc文件最后，增加一行。重启终端后再次输入 mysql -u root -p就可以直接使用了


7、为了弄懂 ~/.zshrc和 ~/.bash_profile的区别，请看这篇文章

8、 知道~/.zshrc和 ~/.bash_profile的区别后，我们环境变量还可以这样配置

9、当配置好环境变量后，mysql/bin下的命令就都可以使用了，我们在任意路径下执行命令：mysql_secure_installation，交互过程中不断输入y或者n，可以修改密码。

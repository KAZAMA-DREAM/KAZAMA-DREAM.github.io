关于SpringCloud Config分布式配置中心中Config客户端配置与测试的相关疑点

一、org.eclipse.jgit.api.errors.TransportException: git@github.com:xx/springcloud-config.git: Auth fail错误

周阳老师的uri是这么写的：**uri: git@github.com:zzyybs/springcloud-config.git**

改成github默认给的这种就好了：**uri: https://github.com/zzyybs/springcloud-config.git**


私有仓库注意要配置账号密码，或者依旧使用ssh，然后多配个私钥

二、SpringCloud Config 使用ssh连接github报错【方案2没试过】：

  • 在 Git Base 中测试连接 ssh -T git@github.com 却不存在问题，并且可以正常的从代码仓库上克隆代码。经过各种尝试最终将排查目标定位到证书的问题上。
  • 之前使用的是 rsa-key 的方式生成的证书，使用的命令是 ssh -keygen -t rsa -C “xxxx@qq.com”, 这种方式生成的证书在 Git Base 上操作是没有问题的，但是在 SpringCloud Config 就是不行。于是我尝试使用以下命令

生成证书，再次在 SpringCloud Config 中使用，问题解决。


可能原因：

  • 第一种方式生成的私钥是以BEGIN RSA PRIVATE KEY开头，END RSA PRIVATE KEY结尾。

  • 第二种方式生成的私钥是以BEGIN OPENSSH PRIVATE KEY开头，END OPENSSH PRIVATE KEY结尾。

    SpringCloudConfig访问git的组件不支持openssh这种格式的秘钥。

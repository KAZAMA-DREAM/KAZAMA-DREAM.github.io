com.mysql.cj.jdbc.exceptions.CommunicationsException: Communications link failure

场景：

通常在新建一个项目是，出现数据库连接异常（communications link failure 表示连接异常）

问题描述：

原因分析：

一般看Caused by:后面的内容，看是什么原因导致的
这里是 Communications link failure 连接失败
Connection refused: connect 拒绝连接

解决方案：

看数据库服务是否启动，window系统用管理员模式打开cmd，输入net start mysql
如果是虚拟机，输入systemctl start mysqld

如果服务已启动，则看连接数据库的配置文件
数据库的URL是否正确，数据库的用户名，数据库的密码是否正确

如果数据库已启动，账号密码都填写正确，url填写正确，看数据库的端口号是否被占用

连接数据库异常原因：

  1. 看数据库是否被启动
  2. 看数据库的配置文件是否填写正确（用户名，密码，url）
  3. 看端口号是否被占用
